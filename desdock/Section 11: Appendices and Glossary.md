# 11. Приложения и глоссарий

---

## 11.1. Термины и понятия

| Термин                                 | Определение                                                                                                              |
| -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| **Заявка (Notice)**                    | Игровая миссия, оформленная как документ. Имеет тип (обычная, дуальная, магограмма) и свой набор возможных исполнителей. |
| **Магограмма (Magogram)**              | Диалоговая сцена особого типа, где игрок читает и выбирает фразы. Может иметь ветвления и последствия.                   |
| **Исполнитель (Executor)**             | Герой, которого игрок назначает на выполнение заявки. Его выбор определяет результат.                                    |
| **Раунд (Day)**                        | Игровой день, состоящий из утренней сцены, выбора заявок и вечернего подведения итогов.                                  |
| **Флаг (Flag)**                        | Булевый параметр, фиксирующий последствия заданий и состояние мира.                                                      |
| **Репутация (Reputation)**             | Отношение мира к гильдии; влияет на тексты газет и возможные реакции NPC.                                                |
| **Золото (Gold)**                      | Ресурс, отражающий финансовое состояние гильдии. Используется как фильтр доступа к заданиям.                             |
| **Хаос (Chaos)**                       | Персональная метрика героя; отражает внутренние изменения и сбои системы.                                                |
| **Газета (Newspaper)**                 | Ежедневное отражение событий мира. Формат — короткие заметки с саркастическим уклоном.                                   |
| **Богиня Хаоса (Chaos Deity)**         | Голос, наблюдающий за игроком. Появляется как шёпот, когда закон даёт сбой.                                              |
| **Гильдия**                            | Главный хаб игры. Здесь происходят утренние диалоги, выбор заданий и чтение газет.                                       |
| **Портрет героя (Character Portrait)** | Визуальное представление персонажа. Меняется в зависимости от стадии хаоса.                                              |
| **Откат (Cycle Reset)**                | Сюжетное событие — повтор дня. Символ хаотического вмешательства в порядок.                                              |
| **Перки (Perks)**                      | Уникальные способности героев, открывающие особые ветки в магограммах.                                                   |
| **RoundSummary**                       | Экран вечерних итогов с конвертами и газетой. Финальная сцена каждого дня.                                               |

## 11.4. Состояния и стадии хаоса

| Уровень | Название   | Внешние проявления                                               | Эффект                                                                   |
| ------- | ---------- | ---------------------------------------------------------------- | ------------------------------------------------------------------------ |
| **H0**  | Норма      | Стандартный портрет, нейтральные реплики.                        | Нет эффектов.                                                            |
| **H1**  | Сбой       | Тени под глазами, фиолетовые отблески. Реплики становятся резче. | Открываются альтернативные ветки в магограммах.                          |
| **H2**  | Деформация | Уверенные цветовые искажения, лёгкие артефакты интерфейса.       | +1 шанс хаоса в заданиях, потеря репутации при законопослушных решениях. |
| **H3**  | Разрыв     | Полностью изменённый портрет, шёпоты в интерфейсе.               | Блокируются “правильные” ветки, открываются уникальные хаос-миссии.      |

## 11.5. Форматы JSON (для справки)

| Файл                 | Назначение           | Ключевые поля                                                   |
| -------------------- | -------------------- | --------------------------------------------------------------- |
| `notices.json`       | Заявки               | `notice_id`, `title`, `type`, `round_id`, `allowed_executors[]` |
| `rules.json`         | Исходы заданий       | `notice_id`, `executor_id`, `rep`, `gold`, `flag`, `chaos`      |
| `rounds_config.json` | Настройки раундов    | `day_id`, `active_notices[]`, `available_characters[]`          |
| `characters.json`    | Герои                | `id`, `name`, `quote`, `traits`, `chaos_level`                  |
| `dialogs.json`       | Диалоги / магограммы | `id`, `text`, `choices[]`, `next_id`, `conditions[]`            |
| `newspaper.json`     | Газеты               | `day_id`, `articles[]`, `headlines[]`                           |

## 11.9. Стиль повествования (для писателей)

* **Тон:** формально-ироничный, как служебная записка, написанная поэтом.
* **Язык:** современный, с редкими старинными оборотами (“согласно пункту”, “в порядке исключения”).
* **Комизм:** не фарс, а контраст между бумажной логикой и живыми эмоциями.
* **Философия:** бюрократия не злая и не добрая — она просто есть.
  И если её рушить, остаётся хаос.
  Если сохранять — медленно гниёшь внутри.

---

## 11.10. Для разработчика

* Любое изменение в JSON должно синхронизироваться с `WorldStateManager` и `RoundSummary`.
* Новые герои добавляются через `characters.json`, портрет — в `Resources/Portraits`.
* Флаги лучше именовать в едином стиле `FLAG_[СУТЬ]_[СЛЕДСТВИЕ]`.
* Внутренняя тестовая команда может использовать консольные команды:

  * `/addrep`, `/addgold`, `/chaos [id]`, `/nextday`.
